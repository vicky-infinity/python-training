Q41. E-Commerce Order Processing System
--------------------------------------------------

Design a mini e-commerce system.

Create the following classes:

- Customer
- Product
- Order

Requirements:

- A Customer can place multiple Orders.
- An Order can contain multiple Products.
- Calculate the final order amount after applying category-wise discounts.
- Log every successful order.
- Raise exceptions for invalid product prices or quantities.
- Display the top 3 customers based on total purchase amount.
- Build a dictionary
  {customer_name: total_spent}.

Q50. University ERP System
--------------------------------------------------

Design a simplified University ERP.

Create the following classes:

- Student
- Professor
- Course
- Enrollment

Requirements:

- Students can enroll in multiple courses.
- Professors can teach multiple courses.
- Validate course enrollment.
- Prevent duplicate enrollments using exception handling.
- Calculate grades for every enrolled student.
- Use decorators to log grade generation time.
- Maintain logs for enrollments and results.
- Generate the following reports:

  • Student-wise grade report

  • Course-wise average marks

  • Professor-wise teaching load

  • Top 5 students based on average marks

  • Dictionary
    {student_name: GPA}

- Save the final reports to separate files.
